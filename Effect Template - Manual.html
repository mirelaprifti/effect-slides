An effect is a description of a computation —
not the computation itself.

Build programs as values. Compose them with confidence. Run them only at the edge of your application.

Three pillars of an Effect program.

Every effect is parameterised by three type variables — together they describe everything the runtime needs to know.

A · SUCCESS

The value an effect produces

Anything from a parsed JSON to an HTTP response — typed and tracked.

E · FAILURE

Typed, recoverable errors

The compiler tracks every failure channel — no surprises at runtime.

R · REQUIREMENTS

Dependencies, made explicit

What the effect needs to run — provided once, at the edge of your program.

ERRORS

Recovery, retry,
fall-back — all explicit.

Effect tracks every failure type in the E channel. The compiler will not let you forget about NotFound.

And because the error path is just another value, you compose it the same way you compose success.
