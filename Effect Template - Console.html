three pillars
07/14

Three pillars of an Effect program.

Every effect is parameterised by three type variables — together they describe everything the runtime needs to know.

A · SUCCESS

Async, concurrency, & resource management

The value an effect produces if it succeeds. Everything from a parsed JSON to a HTTP response.

E · FAILURE

Start by refactoring small parts of your app

Typed errors. The compiler tracks every failure channel — no surprises at runtime.

R · REQUIREMENTS

Integrates seamlessly with OpenTelemetry

Dependencies the effect needs to run — provided once, at the edge of your program.

errors as values
10/14
ERRORS

Recovery, retry, fall-back — all explicit.

Effect tracks every failure type in the E channel. The compiler will not let you forget about NotFound.

And because the error path is just another value, you compose it the same way you compose success.

Note catchTags exhaustively matches the discriminated _tag field.
users.ts TS14 LOC 
const safe = program.pipe(  Effect.catchTags({    NotFound: () =>      Console.log("user not found"),    DbError:   (e) =>      Effect.fail(new UserFault(e))  }),  Effect.retry(Schedule.exponential("100 millis")),  Effect.timeout("3 seconds"),)
